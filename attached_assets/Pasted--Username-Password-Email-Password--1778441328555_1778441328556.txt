צריך לשנות את התחברות האדמין כך שהמשתמש יתחבר עם Username + Password במקום Email + Password, אבל בפועל ההתחברות תמשיך לעבוד דרך Supabase Auth.

פרטי המשתמש:
username במסך האדמין:
lior

email בפועל ב־Supabase Auth:
lior.patisserie@outlook.com

דרישות:
1. בקובץ lior-admin.html:
   - להחליף את שדה האימייל לשדה שם משתמש:
     input id="adminUsername"
     type="text"
     autocomplete="username"
     placeholder="שם משתמש"
   - להשאיר שדה סיסמה:
     input id="adminPassword"
     type="password"
     autocomplete="current-password"
     placeholder="סיסמה"

2. בקובץ admin.js:
   - להסיר לחלוטין כל שימוש ב־TEMP_ADMIN_PASSWORD.
   - ליצור מיפוי פנימי:
     const ADMIN_USERS = {
       lior: "lior.patisserie@outlook.com"
     };

3. בעת לחיצה על loginButton:
   - לקרוא את adminUsername ואת adminPassword.
   - לבצע trim ל־username.
   - להמיר את username לאותיות קטנות:
     username.toLowerCase()
   - לבדוק אם username קיים בתוך ADMIN_USERS.
   - אם לא קיים, להציג:
     "שם המשתמש או הסיסמה שגויים"
   - אם קיים, לקחת את המייל מהמיפוי:
     const email = ADMIN_USERS[username];
   - לבצע התחברות מול Supabase:
     client().auth.signInWithPassword({ email, password })

4. אם ההתחברות הצליחה:
   - להסתיר את loginCard.
   - להציג את adminApp.
   - להריץ initAdmin().

5. אם ההתחברות נכשלה:
   - להציג הודעה:
     "שם המשתמש או הסיסמה שגויים"

6. להוסיף בדיקה בטעינת העמוד:
   - אם יש session פעיל של Supabase Auth, להכניס אוטומטית לאדמין.
   - אם אין session, להשאיר את מסך ההתחברות.

7. להוסיף או להשאיר כפתור התנתקות:
   - id="logoutButton"
   - בלחיצה לבצע:
     client().auth.signOut()
   - להסתיר את adminApp.
   - להציג את loginCard.

8. לא לשים שום סיסמה בקוד.
9. לא להשתמש ב־service_role key.
10. להשתמש רק ב־SUPABASE_URL וב־SUPABASE_ANON_KEY מתוך supabase-config.js.
11. פעולות שמירה והעלאת תמונות באדמין צריכות לעבוד רק למשתמש authenticated.
12. האתר הציבורי צריך להמשיך לקרוא נתונים כרגיל.

- לציין אילו קבצים שונו.
