בקשה אחת לקודקס — הקטנת טקסטים ותיקון תצוגת תמונות באדמין
יש לבצע תיקון ממוקד באזור האדמין, בעיקר באזור עריכת כרטיסי “מה מיוחד אצלנו”.
כרגע הממשק גדול מדי, הטקסטים גדולים מדי, התיבות תופסות יותר מדי מקום, ובאזור התמונות יש מצב לא תקין שבו מוצגת תמונה אבל עדיין כתוב “אין תמונה זמינה”.

1. להקטין משמעותית את כל הטקסטים באדמין
הטקסטים באדמין עדיין גדולים מדי וגורמים לכל התיבות להיות גבוהות, כבדות ולא נוחות לתפעול.
יש להקטין משמעותית:


כותרות שדות כמו:
“כותרת”, “טקסט”, “תמונה”, “מוצג באתר”.


טקסטים בתוך:


input


textarea


select




טקסטים בכפתורים כמו:


“שמירה”


“בחירת קובץ”


“הסתר מהאתר”




כותרות בתוך הכרטיסים.


טקסטים בטאבים העליונים של האדמין.


line-height בכל אזור האדמין.


המטרה היא שהממשק ירגיש קל, קומפקטי ונוח לסריקה, עם פחות גובה לכל תיבה ויותר מידע שנכנס במסך אחד.
כיוון CSS מומלץ:
.admin-page {  font-size: 13px;}.admin-page label,.admin-page .field-label {  font-size: 12px;  line-height: 1.2;}.admin-page input,.admin-page textarea,.admin-page select {  font-size: 13px;  line-height: 1.25;}.admin-page button {  font-size: 13px;  line-height: 1.2;}.admin-page .admin-card-title,.admin-page h3 {  font-size: 15px;  line-height: 1.2;}.admin-page .admin-tabs button {  font-size: 13px;}
לא להשתמש ב־zoom או transform: scale.
צריך לבצע הקטנה אמיתית דרך font-size, line-height, padding, gap, height, width.

2. לצמצם גובה ורוחב של התיבות באדמין
התיבות באזור האדמין גדולות מדי ומקשות על התפעול.
יש לבצע:


לצמצם גובה תיבות.


לצמצם רוחב תיבות.


לצמצם padding פנימי.


לצמצם gap בין תיבות.


לצמצם רווחים בין label לשדה.


לצמצם רווחים בין שדה לשדה.


להקטין textarea שלא יהיה גבוה מדי.


להקטין אזורי תמונה / preview.


להקטין כפתורים, בלי לפגוע בנוחות לחיצה.


לשמור על ממשק קריא ולא צפוף מדי.


המטרה:
שהאדמין יהיה קל לתפעול ולא כבד או מסורבל.

3. אזור “מה מיוחד אצלנו” — 3 תיבות בשורה
באזור האדמין של “מה מיוחד אצלנו” יש להציג את שלושת הכרטיסים בשורה אחת בדסקטופ.
הכרטיסים הקבועים הם:


עבודת יד


טעמים עשירים


חגיגי ומפנק


יש לבצע:


להציג 3 תיבות בשורה אחת בדסקטופ.


שכל תיבה תהיה קומפקטית ואחידה.


שכל התיבות יהיו באותו גובה ורוחב.


לא להציג אותן אחת מתחת לשנייה בדסקטופ.


לצמצם padding ו־gap בתוך כל תיבה.


לאפשר עריכה קלה ומהירה של כל כרטיס.


במובייל אפשר לעבור לתיבה אחת בשורה.



4. להסיר “הוספת כרטיס מידע” מהאזור הזה
באזור “מה מיוחד אצלנו” יש להסיר את הכפתור / הפעולה:
“הוספת כרטיס מידע”
הסיבה:
באזור הזה צריכים להיות רק 3 כרטיסים קבועים בלבד, ואין מקום באתר לכרטיס רביעי.
יש לבצע:


להסיר את כפתור “הוספת כרטיס מידע”.


לא לאפשר יצירת כרטיס רביעי באזור הזה.


להשאיר רק אפשרות לערוך את שלושת הכרטיסים הקיימים.


אם קיימת פונקציית add כללית — לבטל אותה רק עבור “מה מיוחד אצלנו”, בלי לפגוע באזורים אחרים.



5. תיקון באג תמונה — לא להציג “אין תמונה זמינה” כשיש תמונה
באדמין יש באג שבו מוצגת תמונה / preview, אבל עדיין מופיע הטקסט:
“אין תמונה זמינה”
זה לא תקין ומבלבל.
יש לתקן את תנאי ההצגה כך ש:


אם קיימת תמונה או preview תקין — להציג רק את התמונה.


אם קיימת תמונה — לא להציג בכלל את הטקסט “אין תמונה זמינה”.


אם אין תמונה באמת — רק אז להציג placeholder קטן עם “אין תמונה זמינה”.


לא להציג תמונה ו־placeholder יחד.


אחרי בחירת קובץ חדש — הטקסט “אין תמונה זמינה” צריך להיעלם מיד.


אחרי טעינת נתונים מ־Supabase / settings — המצב צריך להתעדכן נכון.


לבדוק שלא נשארת שכבת placeholder מעל התמונה.


אם התמונה קיימת אבל לא נטענת — להציג הודעה אחרת כמו “התמונה לא נטענה”, ולא “אין תמונה זמינה”.


לוגיקה רצויה:
const hasImage = Boolean(imageUrl || previewUrl);if (hasImage) {  showImagePreview();  hideEmptyImagePlaceholder();} else {  hideImagePreview();  showEmptyImagePlaceholder();}

6. בדיקות חובה
לאחר התיקון יש לבדוק:


שהטקסטים באדמין קטנים משמעותית.


שהתיבות נמוכות וקומפקטיות יותר.


שיותר מידע נכנס במסך אחד.


שכרטיסי “מה מיוחד אצלנו” מוצגים 3 בשורה בדסקטופ.


שכפתור “הוספת כרטיס מידע” הוסר מהאזור הזה.


שאין אפשרות להוסיף כרטיס רביעי באזור הזה.


שאם יש תמונה — לא כתוב “אין תמונה זמינה”.


שאם אין תמונה — כן מופיע placeholder מתאים.


שכפתורים לא נחתכים.


שהממשק נשאר קריא ונוח לתפעול.



משפט סיכום לקודקס
יש להקטין משמעותית את כל הטקסטים והתיבות באזור האדמין, במיוחד בכרטיסי “מה מיוחד אצלנו”, כך שהממשק יהיה קומפקטי, קל ונוח לתפעול. יש להציג את שלושת הכרטיסים בשורה אחת, להסיר את האפשרות “הוספת כרטיס מידע”, ולתקן את באג התמונה כך שלא יופיע הטקסט “אין תמונה זמינה” כאשר בפועל קיימת תמונה או preview.
